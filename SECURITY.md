# 安全策略

## 漏洞上报

请不要在公开 Issue 中提交安全漏洞。

请通过以下方式私下联系：
- `support@restaurantiq.ai`

建议附上：
- 影响页面/接口
- 复现步骤
- 潜在影响范围
- 建议修复方式（如有）

## 密钥与敏感信息规则

- 禁止提交真实密钥（API Key、OAuth Secret、私钥等）。
- 真实密钥只存放在环境变量（如 `.env.local`、部署平台 Secrets）。
- 浏览器端仅暴露必要的 `NEXT_PUBLIC_*` 变量。
- 服务端密钥只能在服务端路由/服务端动作中使用。

## 响应时效

我们会在 3 个工作日内确认收到安全报告。
